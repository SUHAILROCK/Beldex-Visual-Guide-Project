What Are Stealth Addresses?

A stealth address ensures that the receiver's public address is never directly exposed on the blockchain.

üîì In Bitcoin

  • ‚ùå The same public address can be reused
  • ‚ùå Anyone can track incoming payments

üîí In Beldex

  • ‚úÖ A new one-time address is created for each payment
  • ‚úÖ Observers cannot link multiple payments to the same recipient

The receiver's actual wallet address remains private at all times.

How It Works

Sender generates a one-time public key derived from the receiver's address. Only the receiver can detect and spend it.

Stealth addresses operate through the following mechanism:

üîë

Public View Key & Spend Key

Each Beldex wallet has a public view key and a public spend key.

These are used to generate one-time addresses for incoming payments.
üì®

One-Time Address Generation

The sender uses the receiver's public keys to mathematically generate a unique one-time address.

Even if the same person receives 100 payments, each address looks completely unrelated.
üîç

Private Scanning

Only the receiver, using their private view key, can detect which outputs belong to them.

No outside observer can link those outputs together or identify the recipient.

Why Beldex Uses Stealth Addresses

  • Protects receiver identity
  • Prevents address reuse tracking
  • Stops wallet balance analysis
  • Strengthens overall privacy
  • Complements Ring Signatures and RingCT

Without stealth addresses, receiver wallets could be tracked.
With stealth addresses, every payment appears unique and unlinkable.
