Ring Signatures Sender Privacy

Mixes the real sender's signature with decoy signatures. No observer can determine which member of the group actually signed the transaction.

Stealth Addresses Receiver Privacy

Every payment creates a unique one-time address. Even hundreds of payments to the same person appear completely unrelated on the blockchain.

  • ‚úî One-time address per transaction
  • ‚úî Payments cannot be publicly linked

One-Time Keys Unlinkability

Each transaction generates a unique one-time public key derived from the receiver's address. This ensures no two outputs can be linked to the same receiver.

  • ‚úî Derived from receiver's public key
  • ‚úî Unique per transaction output
  • ‚úî Cannot be traced back to wallet address
  • ‚úî Works with stealth addresses

Key Image Anti Double-Spend

A unique cryptographic image derived from each transaction output. The network stores all key images and rejects any that have been seen before ‚Äî preventing double-spending without revealing the spender.

  • ‚úî Unique per transaction output
  • ‚úî Deterministic ‚Äî same input always produces same key image
  • ‚úî Cannot be reversed to find the original key
  • ‚úî Essential complement to ring signatures

Overview

RingCT (Ring Confidential Transactions) was introduced in 2017 on Monero.

It hides the transaction amount while still allowing the network to verify everything is correct.

This made Monero's privacy complete by default.
